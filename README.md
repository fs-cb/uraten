# URATEN サイト（MVP）

静的サイト。`data/events.json` を読んでイベントカレンダーを描画する。
JSONは将来「Tally → Sheets → 変換スクリプト」が生成する公開データで、このリポジトリへの git push が公開トリガー。

## 構成

```
index.html        … サイト本体（モック由来。イベント欄のみJSON駆動に配線済み）
data/events.json  … 公開イベントデータ（変換スクリプトの出力先。今はサンプル）
```

## デプロイ手順（初回のみ）

1. GitHubにリポジトリを作成し、この一式を push する
2. Cloudflareダッシュボード → Workers & Pages → 「Pages」→「Gitに接続」
3. リポジトリを選択し、ビルド設定は以下のとおり
   - フレームワークプリセット：**なし（None）**
   - ビルドコマンド：**空欄**
   - ビルド出力ディレクトリ：**/**（ルート）
4. デプロイ完了後、`https://＜プロジェクト名＞.pages.dev` で公開される

以後は git push するだけで自動で再配信される。`ura-ten.jp` 取得後はPagesのカスタムドメイン設定でDNSを向けるだけ（サイト側の変更は不要）。

## ローカル確認の注意

`index.html` をダブルクリックで開くと **fetch が失敗しカレンダーが表示されない**（`file://` ではJSONを読めない）。必ずローカルサーバー経由で見ること。

```
python -m http.server 8000
# → http://localhost:8000
```

## events.json のスキーマ（変換スクリプトとの契約）

```json
{
  "generated_at": "ISO 8601日時",
  "events": [
    {
      "id":    "evt-0001（申請ID＋連番など一意なら形式自由）",
      "name":  "イベント名",
      "date":  "YYYY-MM-DD（必須・ISO形式）",
      "time":  "HH:MM（任意）",
      "venue": "会場名（任意）",
      "area":  "エリア（任意・例：天神）",
      "url":   "告知ページURL（任意。あれば行全体がリンクになる）"
    }
  ]
}
```

- 過去日のイベントはフロント側で自動的に非表示になる（並びは日付昇順）
- 審査メモ・主催者連絡先など運営用の情報は**このJSONに絶対に入れない**（Sheetsに留める）
